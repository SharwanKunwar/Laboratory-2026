## 1. Information System

### Q1. What is an Information System?

An **Information System (IS)** is a system that **collects, processes, stores, and distributes information** to support decision-making and control in an organization.

### Q2. Components of Information System

* Hardware
* Software
* Data
* People
* Procedures

---

## 2. SDLC (System Development Life Cycle)

### Q3. What is SDLC?

**SDLC** is a step-by-step process used to **develop information systems efficiently and systematically**.

### Q4. Phases of SDLC

1. Planning
2. System Analysis
3. System Design
4. Implementation
5. Testing
6. Deployment
7. Maintenance

---

## 3. CASE Tools

### Q5. What are CASE Tools?

**CASE (Computer Aided Software Engineering)** tools are software tools that help in **designing, analyzing, and maintaining software systems**.

### Q6. Advantages of CASE Tools

* Improves productivity
* Reduces errors
* Saves time and cost
* Provides better documentation

---

## 4. RAD (Rapid Application Development)

### Q7. What is RAD?

**RAD** is a software development model that focuses on **fast development and quick delivery** using prototypes.

### Q8. Features of RAD

* Rapid development
* User involvement
* Reusable components
* Prototype-based approach

---

## 5. Agile Development

### Q9. What is Agile Development?

**Agile** is a development approach that focuses on **incremental development**, flexibility, and customer collaboration.

### Q10. Principles of Agile

* Customer satisfaction
* Continuous delivery
* Adapts to changes
* Team collaboration

---

## 6. Project Management

### Q11. What is Project Management?

Project Management is the process of **planning, organizing, and controlling resources** to achieve project goals within time and budget.

### Q12. Functions of Project Management

* Planning
* Scheduling
* Monitoring
* Controlling

---

## 7. Project Scheduling

### Q13. What is Project Scheduling?

Project scheduling is the process of **deciding start and finish times** of project activities.

### Q14. Tools Used in Project Scheduling

* Gantt Chart
* PERT Chart
* CPM (Critical Path Method)

---

## 8. System Analyst

### Q15. Who is a System Analyst?

A **System Analyst** analyzes user requirements and designs system solutions to meet business needs.

### Q16. Roles of a System Analyst

* Requirement analysis
* System design
* Acts as a bridge between users and developers
* Documentation

---

## 9. Important Short Notes

### Q17. Difference between SDLC and Agile

* **SDLC** follows a fixed sequence of phases
* **Agile** is flexible and iterative

### Q18. Importance of Information System

* Better decision making
* Improves efficiency
* Supports business operations
