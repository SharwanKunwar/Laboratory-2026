## 1. what is meant by baseline projet plan ? Explain major parts of baseline project plan.

A **Baseline Project Plan (BPP)** is the initial approved plan for a project that describes how the project will be carried out. It provides a reference point for managing and evaluating the project’s progress in terms of **scope, cost, and schedule**.

---

## Major Parts of Baseline Project Plan

### 1. Project Introduction
This section provides a brief overview of the project, including:

- Project background  
- Purpose of the project  
- Objectives  
- Problem the system aims to solve  

---

### 2. System Description
This part describes the proposed system and explains:

- Major features of the system  
- Key functions  
- How the system will operate  

---

### 3. Feasibility Assessment
This section evaluates whether the project is practical and achievable. It includes:

- **Technical Feasibility** – Availability of required technology and skills  
- **Economic Feasibility** – Cost effectiveness of the project  
- **Operational Feasibility** – Whether the system will work in the organization  
- **Schedule Feasibility** – Whether the project can be completed on time  

---

### 4. Management Issues
This section explains how the project will be managed. It includes:

- Project team roles and responsibilities  
- Communication methods  
- Project schedule  
- Risk management  
- Monitoring and control of project activities

## 3.Define system testing. Write and explain types of system tets.

**System testing** is the process of testing the complete and fully integrated system to verify that it meets the specified requirements. It ensures that the system works correctly as a whole before it is deployed to users.

System testing checks **functionality, performance, security, and usability** of the system.

---

## Types of System Testing

### 1. Performance Testing
This test evaluates how the system performs under expected workload conditions.

**Example:**  
Checking how many users can simultaneously log in to an online shopping website without slowing down.

---

### 2. Security Testing
Security testing ensures the system is protected against unauthorized access, data breaches, and vulnerabilities.

**Example:**  
Testing user authentication, password protection, and data encryption in a banking application.

---

### 3. Stress Testing
Stress testing examines how the system behaves under extreme or peak conditions beyond normal operational capacity.

**Example:**  
Submitting thousands of requests to a server in a short time to see if it crashes.

---

### 4. Usability Testing
Usability testing checks how easy and user-friendly the system is for the end-users.

**Example:**  
Testing whether users can easily navigate a mobile app to find products and complete purchases.

---

### 5. Acceptance Testing (Optional)
Acceptance testing verifies if the system meets the business requirements and is ready for deployment.

**Example:**  
A client testing a custom CRM system to confirm it satisfies their operational needs before going live.

## 4. "Documentation play vital role in system development and review" justy this stement

## Statement
**Documentation plays a vital role in system development.** It records all necessary information about a system and supports understanding, development, and maintenance throughout the system’s lifecycle.

---

## Justification

### 1. Facilitates Understanding
Documentation provides a clear description of the system’s design, structure, and functionality, helping developers and stakeholders understand how the system works.

**Example:**  
Design diagrams, flowcharts, and user manuals make it easier for new developers to understand the system.

---

### 2. Assists Maintenance
Well-documented systems are easier to maintain, update, and fix. It helps developers identify errors and implement changes without breaking the system.

---

### 3. Supports Training
Documentation helps train new users and staff on how to operate the system effectively.

**Example:**  
User guides and help manuals provide step-by-step instructions for using software applications.

---

### 4. Ensures Consistency
Proper documentation ensures consistency in development, testing, and deployment, reducing errors and misunderstandings.

---

### 5. Provides Reference for Future Development
Documentation serves as a reference for upgrading or expanding the system in the future.

---

**Conclusion:**  
Documentation is essential in system development because it improves communication, ensures proper system understanding, simplifies maintenance, and supports future growth.

## 5. Why system maintenance is importaint? Explain different types of maintenance.

System maintenance is important because after a system is implemented, it **needs to be updated, corrected, and improved** to remain useful, efficient, and aligned with changing user requirements or technology.

Reasons for system maintenance:  
- Fixing errors or bugs  
- Improving system performance  
- Adapting to new hardware or software  
- Meeting changing business or user needs  

---

## Types of System Maintenance

### 1. Corrective Maintenance
Fixes errors or defects discovered after the system is deployed.

**Example:**  
Correcting a calculation error in a payroll system.

---

### 2. Adaptive Maintenance
Modifies the system to work in a **new environment** or with updated technology.

**Example:**  
Updating software to be compatible with a new operating system.

---

### 3. Perfective Maintenance
Enhances system performance or adds **new features** based on user feedback.

**Example:**  
Adding a search filter feature in an e-commerce application.

---

### 4. Preventive Maintenance
Improves system reliability and prevents future problems.

**Example:**  
Refactoring old code to reduce the risk of system failure later.

---

**Conclusion:**  
System maintenance is crucial for keeping systems **functional, efficient, and up-to-date**, ensuring long-term usability and reliability.

## 6. What is normalization ? Explain the mojor objectives of database desing.

**Normalization** is the process of organizing data in a database to **reduce redundancy and improve data integrity**. It involves dividing large tables into smaller, related tables and defining relationships between them.

**Purpose:**  
- Avoid duplicate data  
- Ensure data consistency  
- Simplify database maintenance  

---

## Major Objectives of Database Design

### 1. Reduce Data Redundancy
Eliminate storing the same data multiple times to save space and prevent inconsistencies.

### 2. Improve Data Consistency
Ensure that all copies of the data are accurate and up-to-date across the database.

### 3. Enhance Data Integrity
Maintain the correctness, accuracy, and reliability of the data.

### 4. Efficient Data Storage
Organize data efficiently to make optimal use of storage space.

### 5. Easy Data Maintenance
Make it simpler to insert, update, delete, and manage data in the database.

---

**Conclusion:**  
Normalization and proper database design help build **efficient, reliable, and easy-to-maintain databases**, ensuring data accuracy and reducing storage issues.
